# 技術文書作成ガイドライン

## JAISTの技術文書作成ガイドライン

このガイドラインは、北陸先端科学技術大学院大学（JAIST）の上原隆平教授による技術文書作成の指針です。

### 基本原則

**技術文書は読者に情報を明確に伝えることが目的**

技術文書の目的は、読み手に必要な情報を正確かつ効率的に伝えることです。論理的な構造を保ち、対象読者に適した文体を使用することが重要です。

### 主要セクション

#### 1. 準備（Preparation）

技術文書を書く前の準備段階：

- **テーマ（Theme）**: 明確な執筆目的を設定する
- **標準的な文（Standard sentences）**: 一貫性を保つために模範となる文章を使用する
- **読者（Reader）**: 対象読者の知識レベルを考慮する
- **内容（Content）**: KJ法などを使って情報を体系的に整理する

#### 2. 文の構造と組立（Document Structure）

文書は「木構造」に従うべきであり、自然に流れるように構成します：

- 文書の冒頭で主要なアイデアを明確に提示
- 読者が全体像を把握してから詳細に入れるよう、支援的な詳細を配置
- 「読み手が文の構造を容易に理解できるよう工夫すること」

#### 3. パラグラフ（Paragraphing）

効果的なパラグラフの構成：

- 各段落の冒頭にトピックセンテンスが必要
- トピックセンテンスは簡潔で主題に関連性がある
- 支援文で必要な詳細とコンテキストを提供
- パラグラフは論理的にグループ化する
- トピック指示子で文章を開始する

#### 4. 文の流れ（Text Flow）

情報の論理的な進行：

- 一般的な情報から具体的な情報へ進む
- 文章は論理的に接続する必要がある（急な転換は読者を混乱させる）
- 「逆向きの木」（reverse tree）図構造を使用して適切な情報階層を示す
- アイデアを明確に接続する

#### 5. 文体（Style）

一貫性のあるスタイルの維持：

- 文書全体で口調と用語の一貫性を保つ
- フォーマルとカジュアルなレジスターを混在させない
- 関連するアイデアには並列構造を使用
- 創造的な代替案よりも標準的な表現を使用
- 一貫した音声とトーンを維持

#### 6. 事実と意見（Facts and Accuracy）

正確性と根拠：

- 事実は検証が必要
- 曖昧な表現を避ける
- 不明確な先行詞の参照を避ける
- 意味を曖昧にする過度に複雑な構造を避ける
- 事実の記述と解釈を明確に区別する
- 適切な引用と一貫した事実で信頼性を構築

#### 7. 簡潔・明確な表現（Clarity Techniques）

明確性を高める技術：

- 曖昧な表現を排除する
- 暗黙の仮定よりも明示的な言語を使用
- 略語を適切に定義する
- 冗長な表現を避ける
- 技術用語を正確に定義する
- 略語を一貫して使用する

#### 8. 正確な表現（Common Errors）

避けるべき一般的なエラー：

- 不明確な代名詞の参照
- 主語と述語の不一致
- 扱いにくい文構造
- 文法エラー
- 主語と動詞の整合性の問題
- 冗長性

#### 9. 実習（Practice Examples）

実践的な改善例：

- リストラクチャリングが可読性と理解をどのように改善するかを示す複数の修正前・修正後の例
- 明確さと構造における効果的な改訂技術のデモンストレーション

### 推奨される執筆プロセス

1. **準備段階**: テーマを確立し、読者を理解し、内容を整理する
2. **構造設計**: 論理的な流れを持つ明確な文書構造を作成
3. **執筆**: 一貫したスタイルで明確かつ正確に書く
4. **推敲**: エラーをチェックし、明確性を向上させ、流れを改善する

### まとめ

効果的な技術文書は以下の特徴を持つ：
- 明確な目的と対象読者
- 論理的な構造と流れ
- 一貫したスタイルと用語
- 正確で検証可能な情報
- 簡潔で明確な表現
- 適切な段落構成

このガイドラインに従うことで、読者にとって理解しやすく、有用な技術文書を作成することができます。
